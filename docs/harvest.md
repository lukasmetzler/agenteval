# Harvesting from Git History

The `harvest` command scans your git history for commits that involved AI coding tools, then generates task definition YAML files from those commits. The merged diff becomes the "ground truth" for what a good result looks like.

This lets you build eval benchmarks from real work your team has already done, instead of writing synthetic test cases by hand.

## When to Use This

- You've been using AI coding tools (Claude Code, Copilot, Cursor, etc.) and want to build benchmarks from that work
- You want to test whether a change to your CLAUDE.md actually makes your agent better at the tasks your team does
- You want a quick overview of how much AI-assisted work is in your repo
- You're setting up agenteval for the first time and need a starting dataset

## How It Detects AI Commits

agenteval looks for three signals in each commit:

| Signal | How it works | Confidence |
|--------|-------------|------------|
| **Co-Authored-By trailer** | Checks the `Co-authored-by` git trailer for known AI tool emails (`noreply@anthropic.com` for Claude, `noreply@github.com` with "copilot" in the name, etc.) | 0.9 (high) |
| **Author email** | Checks if the commit author email matches known bot patterns | 0.8 |
| **Commit message** | Checks for patterns like "Generated by", bot emojis, etc. | 0.6 (lower) |

When multiple signals match the same commit, the highest confidence wins. Commits below the confidence threshold (default: 0.5) are filtered out.

### Supported AI Tools

- Claude Code (Co-authored-by: `noreply@anthropic.com`)
- GitHub Copilot (Co-authored-by: `noreply@github.com` with "copilot" in name)
- Cursor (email containing "cursor")
- Devin (email containing "devin")
- Aider (email containing "aider")

## Basic Usage

### Preview What's in Your Repo

Start with `--dry-run` to see what agenteval finds without writing any files:

```bash
agenteval harvest --dry-run
```

Output:

```
  Harvest Dry Run
  ═══════════════

  Commits scanned:   142
  AI commits found:  37

  Detected tasks:
    • harvest-abc123d
    • harvest-def456a
    • harvest-789bcd0
    ...

  Skipped: 1
    • 5c637b8: no parent commit
```

### Generate Task Files

```bash
# Write task YAML files to the default directory (tasks/harvested/)
agenteval harvest

# Write to a custom directory
agenteval harvest --output my-benchmarks/
```

Each detected commit becomes one YAML file. For example, `tasks/harvested/harvest-abc123d.yaml`:

```yaml
name: harvest-abc123d
description: "feat: add user authentication module"
prompt: "Add user authentication module"
harness: auto
timeout: 300
assertions:
  - type: files-changed
    pattern: src/auth.ts
  - type: files-changed
    pattern: tests/auth.test.ts
scoring:
  correctness: 0.5
  precision: 0.3
  efficiency: 0.1
  conventions: 0.1
```

### Filter by Date

```bash
# Only scan commits from the last 3 months
agenteval harvest --since 2026-01-01

# Scan a specific date range
agenteval harvest --since 2026-01-01 --until 2026-03-01
```

### Scan a Single Commit

```bash
agenteval harvest --commit abc123def
```

## All Flags

| Flag | Default | Description |
|------|---------|-------------|
| `--since <date>` | all history | Only scan commits after this date (any git date format) |
| `--until <date>` | now | Only scan commits before this date |
| `--commit <hash>` | | Scan a single commit instead of the full log |
| `--output <dir>` | `tasks/harvested` | Directory to write task YAML files |
| `--dry-run` | `false` | List detected commits without writing files |
| `--force` | `false` | Overwrite existing task files (by default, existing files are skipped) |
| `--format <type>` | `yaml` | Output format: `yaml` (task files) or `json` (stdout) |
| `--harness <name>` | `auto` | Set the `harness` field in generated tasks |
| `--timeout <seconds>` | `300` | Set the `timeout` field in generated tasks |
| `--min-confidence <n>` | `0.5` | Minimum detection confidence (0 to 1). Lower = more commits found, but more false positives |
| `--config <path>` | auto-discover | Path to `agenteval.yaml` |

## Using Harvested Tasks

Once you have task YAML files, you can run them through the eval pipeline:

```bash
# Run a single harvested task
agenteval run --task tasks/harvested/harvest-abc123d.yaml

# Run with a specific instruction file to test
agenteval run --task tasks/harvested/harvest-abc123d.yaml --instructions CLAUDE-v2.md
```

The real power is comparing instruction versions:

```bash
# Run with your current instructions
agenteval run --task tasks/harvested/harvest-abc123d.yaml
# Note the run ID, e.g., run-20260401-143022

# Edit your CLAUDE.md...

# Run again with the updated instructions
agenteval run --task tasks/harvested/harvest-abc123d.yaml
# Note the second run ID, e.g., run-20260401-150000

# Compare the two runs
agenteval compare run-20260401-143022 run-20260401-150000
```

## Prompt Quality

The generated prompts are starting points, not production quality. They're derived from commit messages, which describe what was done ("added auth module") rather than what needs doing ("add an auth module"). agenteval converts past-tense to imperative ("added" becomes "Add") and strips conventional-commit prefixes ("feat:", "fix:", etc.).

For best results, review the generated YAML and refine the `prompt` field to match how you'd actually describe the task to an AI agent.

## Idempotency

Running `harvest` multiple times is safe. If a file like `harvest-abc123d.yaml` already exists, it's skipped. Use `--force` to regenerate all files.

## JSON Output

For scripting and pipelines:

```bash
agenteval harvest --format json
```

Returns a JSON object with counts, detected tasks, and skipped commits.

## Configuration

These defaults can be set in `agenteval.yaml` so you don't have to pass them every time:

```yaml
harvest:
  outputDir: "tasks/harvested"    # Where to write task files
  minConfidence: 0.5              # Detection threshold (0-1)
  defaultHarness: auto            # Harness to set in generated tasks
  defaultTimeout: 300             # Timeout to set in generated tasks
```

CLI flags always override config values.

## Edge Cases

- **Initial commit**: Skipped (no parent to diff against)
- **Merge commits**: Uses the first parent diff (matches squash-merge workflows)
- **Empty diffs**: Skipped
- **No AI commits found**: Prints a helpful message suggesting `--min-confidence 0.3`
- **File renames**: Correctly extracts the destination path

## Exit Codes

| Code | Meaning |
|------|---------|
| 0 | Completed (even if no AI commits found) |
| 2 | Error (not a git repo, git not found, etc.) |
